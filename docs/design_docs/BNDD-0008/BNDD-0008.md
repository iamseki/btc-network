## Purpose

This document remains only as historical context.

It previously guided a crawler design that wrote compressed recovery frontiers
into `crawler_run_recovery_points` and attempted startup recovery of the latest
active run after a crash.

That is no longer project direction.

## What Changed

The crawler now uses simpler runtime semantics:

- `crawler_run_checkpoints` remains append-only progress history
- `node_observations` remains append-only observation history
- crawler startup always begins a fresh run from seed nodes
- crash or manual restart means operators simply run crawler again

The shared crawler no longer reads or writes `crawler_run_recovery_points`.

## Why It Was Deprecated

Durable crash recovery did not pay for its complexity in this repository.

Main tradeoffs:

- crawler runs are sporadic, not continuous service workloads
- restarting from scratch is acceptable for current operator workflow
- recovery payload writes added storage growth on every checkpoint interval
- startup recovery added more repository API, SQL, tests, and failure modes
- duplicated observations after restart are acceptable for current research use

Given those constraints, durable recovery was overengineering.

## Current Direction

Current pragmatic choice:

- keep crawler runtime easy to operate and easy to maintain
- prefer fresh-start reruns over partial-run crash recovery
- keep checkpoints useful for visibility, not restart orchestration
- revisit durable recovery only if long-running or costly crawls make restart
  loss a real operational problem

If durable recovery becomes valuable later, it should come back behind a new
BNDD with evidence from real crawler operation rather than speculative
resilience goals.

## Historical Note

Existing databases may still contain the historical
`crawler_run_recovery_points` table because migrations stay forward-only.

That table is now legacy schema baggage:

- it may still be present
- current crawler code does not depend on it
- new work should not extend or rely on it
